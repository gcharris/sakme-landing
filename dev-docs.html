System Inventory

What the engine contains, organized by domain.

Architecture

Architecture Overview

FastAPI backend, 308+ services, 18 output sinks, React frontend with Capacitor mobile wrapping, Telegram integration for ops

Data Model

76 SQLAlchemy models organized by domain. Full enum coverage, relationship mapping, 80+ Alembic migrations with pre-flight validation gates

Event Sourcing & Outbox

Immutable event log with Transactional Outbox Pattern. OutboxRelay for async processing. HITL approval gates prevent unwanted external actions

Multi-Tenancy

Row-level PostgreSQL isolation, Cypher-level Neo4j filtering, tenant context middleware. Complete data separation per creator

Auth System

Four-tier auth hierarchy (Director Key, Dev Token, Dev Stub, Firebase). Multi-provider OAuth (Google, GitHub, Telegram). BYOK API key vault

Core Creative Systems

Learning Engine

How sources become ingredients: interview sessions, multi-source extraction pipeline, story bible generation with character/world/theme decomposition

Knowledge Graph

Neo4j integration with GraphRAG queries. Entity relationships, Flash-to-Foundry triplet seeding from NotebookLM, contradiction detection

Sovereign Codex

Three-layer knowledge stack: Sovereign (director-blessed) > Project (local overrides) > External (imported). Crystallization promotes raw material with blessing scores

Voice Calibration

Tournament-based voice learning. Temperature strategies across [0.7, 0.9] range. VoiceProfile generation per character. Multi-character ensemble calibration

Narrative Governor

Deterministic kill switches (Arc Wholeness, Character Integration, Setup-Payoff Distance). N1-N5 violation taxonomy with severity routing. ORA enforcement loop

Society of Thought

Tournament architecture based on Kim et al. research. Multi-variant generation, consensus voting, wildcard injection (Move 37). Empirically validated: 40% peak capture rate

Creative Refinement Loop

Protocol-based generate-critique-revise cycle. Any Generator + Critic pair plugs in via structural subtyping. Convergence config with per-dimension thresholds

Creator Development Engine

19 decision event types across 5 services. Growth edge detection, provenance scoring, the Mirror (creative self-reflection). Tracks how creators develop over time

Agent & Routing

Agentic Loop

Multi-tool loop runtime with retries, context compaction, fallback strategies, and streaming events. Two-agent architecture: Director + User agents with distinct souls

Prompt Assembler

Three-layer system prompt builder: Soul (identity) + State (pipeline context) + Policy (tool rules). Model-agnostic output works with Gemini, Claude, GPT via BYOK

Pipeline Context Builder

Assembles full pipeline state per tenant per turn (10 parallel DB queries, ~50ms). Stage detection from ONBOARDING through EXPORT. Agent always knows where you are

Infrastructure

Testing

310+ test files, ~2,650 tests. MockLLMClient for deterministic testing, async fixtures, focused domain testing. Merge gates enforce compilation + test + type checks

Deployment

Cloud Run on GCP. Dockerized backend with GitHub Actions CI/CD. Firebase Hosting for frontend. Autonomous deployment runbook with migration pre-flight gates

BYOK Architecture

Bring Your Own Key model. Users supply their own LLM API keys. Platform provides the intelligence layer (soul + state + policy). BudgetOrchestrator with credit ledger

Billing Infrastructure

Provider-agnostic BillingService interface. Paddle adapter live. Add any provider by implementing one ~150 LOC adapter. TierGate middleware enforces subscription levels

Scheduler & Health

APScheduler with PostgreSQL job store for persistence. Daily health checks (DB + Neo4j + LLM providers). Weekly summary aggregation. Telegram alerts on failure

Video Pipeline

5 video generation sinks (Higgsfield, Kling, Veo, Moltbook, Aggregator). SinkRouter for provider selection. Shot planning, tournament scoring, Vertex AI audio isolation

Key Design Decisions

Architectural choices with reasoning.

REASONING

Society of Thought

Multi-variant tournament generation outperforms single-shot prompting. SOT-001 experiment: 40% peak capture rate. 3 variants, 0.2 wildcard ratio, [0.7, 0.9] temperature range. Empirically validated, not theoretical.

GOVERNANCE

Narrative Governor

Deterministic kill switches because LLMs cannot reliably self-govern. N1-N5 violation taxonomy with severity routing. The Governor is the moat — anyone can call Gemini, but consistent creative output requires external governance.

KNOWLEDGE

Three-Layer Codex

Sovereign > Project > External trust hierarchy. Director-blessed knowledge takes precedence. Crystallization promotes raw material with blessing scores. Every creative session builds the knowledge base.

CONSISTENCY

Transactional Outbox

Atomic writes across PostgreSQL + Neo4j + EventStore without 2PC. OutboxRelay processes pending entries asynchronously with retry logic and HITL approval gates. Chosen over Saga pattern for simplicity.

BUSINESS MODEL

Open-Core Split

Seitiate (AGPL engine, free) + Sakme (beautiful app, paid). Monetization boundary is experience, not capability. The engine is identical. Free users wire it themselves; paid users get the effortless experience.

ARCHITECTURE

Entity Component System

The Talent Taxonomy emerged as an ECS — Skills are entities, metadata groups are components, executors are systems. Nobody designed it that way; it was discovered. Creative physics, not gamification.
